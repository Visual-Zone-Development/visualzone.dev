Overview

Context Recall saves your entire workspace state — open files, cursor positions, scroll positions, and Git branches — as named task profiles. Switch between tasks in seconds without losing where you were. Stop wasting time remembering which files you had open or which branch you were on.

Key Features

📋

Task Profiles

Create named profiles that capture your full workspace state with one command.

🔄

Seamless Task Switching

Restore any saved profile instantly — files, cursors, and all.

🌿

Smart Git Integration

Automatically stash or restore uncommitted changes when switching profiles.

⚙️

Flexible Automation

Choose manual, prompt, or fully automatic stash/restore behaviour per profile.

📝

Profile Customization

Rename, edit, and reorder profiles to match your workflow.

🔍

Tree View Management

A dedicated sidebar panel to view, switch, and manage all your profiles.

Use Cases

🔧

Multi-Task Development

Jump between a bug fix, a new feature, and a refactor without losing context on any of them.

👀

Code Reviews

Review a colleague's PR then return to your own work exactly where you left off.

📚

Learning New Codebases

Save reference points as you explore an unfamiliar project.

🌲

Feature Branch Workflow

Each profile maps to a branch — switch branches and workspace state in one action.

Getting Started

  1. Install Context Recall from the VS Code Marketplace and reload VS Code.
  2. Open the Command Palette (Ctrl+Shift+P) and run Context Recall: Create Profile to save your current workspace state.
  3. Switch between profiles from the sidebar panel or via the Command Palette at any time.
